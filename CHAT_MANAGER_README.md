## ✨ 主要功能

### 1. 对话记录查看

#### 功能特性
- 📅 按日期筛选对话
- 📱 按设备筛选对话
- 💬 对话式展示（用户消息 + AI回复）
- ⏰ 显示对话时间戳
- 📊 实时统计（总对话数、今日对话、设备数量）

#### 使用方法
1. 选择日期（默认今天）
2. 选择设备（默认当前设备）
3. 点击"查询"按钮
4. 浏览对话记录

### 2. 每日总结管理

#### 功能特性
- ✨ 手动生成每日总结报告
- 👁️ 在线预览报告内容
- 📥 下载HTML格式报告
- 🔄 重新生成已有报告
- 📊 显示报告统计信息（对话数、生成状态）

#### 使用方法

**生成报告**:
1. 选择日期
2. 选择设备
3. 点击"生成报告"按钮
4. 等待AI分析完成

**查看报告**:
1. 在报告列表中找到目标报告
2. 点击"查看"按钮
3. 在弹出窗口中预览完整报告

**下载报告**:
1. 在报告列表中找到目标报告
2. 点击"下载"按钮
3. 保存为HTML文件

## 🔧 技术架构

### 前端技术
- **纯HTML + CSS + JavaScript** - 无需构建工具
- **响应式设计** - 适配桌面和移动设备
- **渐变背景** - 美观的紫色渐变主题
- **卡片式布局** - 清晰的信息层次

### 后端API

#### 对话相关API
```python
GET  /api/chats/<device_mac>              # 获取对话记录
     ?date=YYYY-MM-DD                     # 按日期筛选
     &limit=100                           # 数量限制
     &offset=0                            # 偏移量

GET  /api/chats/stats                     # 获取统计信息
     ?date=YYYY-MM-DD                     # 日期筛选
     &device_mac=XX:XX:XX:XX:XX:XX       # 设备筛选
```

#### 报告相关API
```python
POST /api/reports/generate                # 生成报告
     {"device_mac": "XX:XX:XX:XX:XX:XX",
      "date": "YYYY-MM-DD"}

GET  /api/reports/<device_mac>/<date>     # 获取报告HTML

GET  /api/reports/<device_mac>/<date>/json # 获取报告元数据

GET  /api/reports/list                    # 获取报告列表
     ?device_mac=XX:XX:XX:XX:XX:XX
     &limit=30
```

## 🔍 常见问题

### Q1: 页面无法加载？
**A**: 检查服务器是否启动：
```bash
ssh root@192.140.190.183
ps aux | grep web_server
```

### Q2: 对话记录显示为空？
**A**: 确认数据库中有数据：
```bash
sqlite3 data/reminders.db 'SELECT COUNT(*) FROM chat_messages'
```

### Q3: 报告生成失败？
**A**: 检查DeepSeek API配置：
```bash
grep LLM_API_KEY web_server.py
```

### Q4: 如何添加新设备？
**A**: 设备会自动注册到数据库，首次对话时会自动创建。
