# 摄像头语音反馈优化

## 📸 改进说明

### 问题描述
用户让机器人拍照识别物体时，机器人直接拍照没有说话，用户不知道机器人在处理什么。

### 解决方案
在 MCP 工具描述中添加了明确的提示，要求 AI 在拍照前先回应。

---

## 🎯 现在的行为

### 用户说：
```
"帮我看一下这是什么"
```

### 机器人会先回应：
```
"好的，稍等我帮你看看"
```
或
```
"好的，我来看看这是什么"
```
或
```
"没问题，我来帮你识别一下"
```

### 然后再拍照并分析

---

## 📝 可用的语音回应

AI 会自动选择以下任一回应：

1. **"好的，稍等我帮你看看"**
2. **"好的，我来看看这是什么"**
3. **"没问题，我来帮你识别一下"**

---

## 🔧 技术实现

### 修改位置
- **文件**: `main/mcp_server.cc`
- **工具**: `self.camera.take_photo`

### 关键修改
在工具描述中添加了 **IMPORTANT** 提示：

```
IMPORTANT: Before taking the photo, you MUST acknowledge the user's request
with a friendly response such as:
  - '好的，稍等我帮你看看'
  - '好的，我来看看这是什么'
  - '没问题，我来帮你识别一下'
```

### 示例对话

| 用户 | AI 回应 | 然后执行 |
|------|---------|---------|
| "帮我看一下这是什么" | "好的，稍等我帮你看看" | 拍照并分析 |
| "这是什么" | "好的，我来看看这是什么" | 拍照并分析 |
| "识别一下这个物体" | "没问题，我来帮你识别一下" | 拍照并分析 |

---
